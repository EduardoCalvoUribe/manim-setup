# FAQ: Getting Help

> Source: https://docs.manim.community/en/stable/faq/help.html

---

## How do I animate X? Why do I get error Y? Can someone help me?

Before asking the community, verify your issue isn't already addressed in the FAQ. Search using your favorite search engine for blog posts, StackOverflow questions, or r/manim subreddit discussions.

When preparing your question, explain your struggle clearly so others can help efficiently.

### Communication Channels

| Channel | Notes |
|---------|-------|
| **Discord** | Most active community; browse `#manim-help` channels and search existing discussions |
| **StackOverflow** | Monitor questions tagged with `manim` |
| **Reddit** | Post to r/manim; Discord or StackOverflow may be better |
| **GitHub** | Start a new discussion if you prefer this platform |

In all channels, follow Manim's Code of Conduct: be friendly, patient, considerate, and respectful.

---

## What should I do if nobody answers my question?

Improve your question by including all relevant information: full error stack traces, rendering code, and commands used. Provide minimal reproducible examples rather than lengthy code blocks.

If your Discord question gets buried, you may ping the `@Manim Helper` role — but avoid this on your first post, as it's considered impolite. Don't spam your question across multiple channels.

Remember: "our community members are volunteers." Quick answers aren't guaranteed if nobody knows the solution, your question lacks clarity, or helpers are occupied elsewhere.

---

## The library does not behave as documented, or something broke in a new release. What should I do?

You've likely found a bug. Check "our list of known issues and feature requests in our GitHub repository." Use the search function and review closed issues, as your problem may already be resolved.

**When submitting an issue:**

1. Verify you're running the latest released version by checking the Changelog.
2. Select the appropriate category (bug report, feature request, or installation issue) when creating a new issue.
3. Prepare a minimal example illustrating the problem.
4. Include the "full terminal output, from the command you used to run the library up to and including the last line with the error message." Include additional log files if mentioned in the error.

---

## I have an idea for a really cool feature. Where should I share my idea?

Submit new suggestions by "creating a new discussion" in the "Suggestions and Proposals" category in the GitHub repository. Don't create feature request issues directly — they will be closed. Once your idea becomes concrete and gains community support, a corresponding issue will be created.
